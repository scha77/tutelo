# T01: Add ShadCN Calendar component and unit tests for override precedence logic

**Installed ShadCN Calendar, extracted `getSlotsForDate` to `src/lib/utils/slots.ts` with override-wins-recurring precedence, and added 6 passing unit tests.**

## What Happened

1. Installed ShadCN Calendar component via `npx shadcn@latest add calendar` ‚Äî created `src/components/ui/calendar.tsx` and added `react-day-picker@^9.14.0` to dependencies.
2. Created `src/lib/utils/slots.ts` with the `getSlotsForDate` pure function extracted from `BookingCalendar.tsx`. Added an optional `overrides` parameter (`AvailabilityOverride[]`). The precedence logic: if any override row matches the date's `YYYY-MM-DD` string, only override-derived slots are returned (recurring is fully ignored); otherwise falls back to recurring day-of-week matching. Exported `AvailabilitySlot`, `AvailabilityOverride`, and `TimeSlot` interfaces.
3. Created `tests/unit/override-precedence.test.ts` with 6 tests: (a) no override ‚Üí recurring, (b) override exists ‚Üí only override slots, (c) zero usable override windows ‚Üí empty even with recurring, (d) overrides for other dates don't affect, (e) override slots sorted by startRaw, (f) no recurring + no overrides ‚Üí empty.
4. Updated `BookingCalendar.tsx` to import `getSlotsForDate` and types from `slots.ts`, removed inline function and interfaces, removed unused `date-fns-tz` imports. Passes empty `[]` for overrides to maintain backward compatibility.
