Fix 1 — `src/app/account/page.tsx` line 77:

Change:
```tsx
const rebookUrl = `/${teacher.slug}#booking?subject=${encodeURIComponent(booking.subject)}`
```
To:
```tsx
const rebookUrl = `/${teacher.slug}?subject=${encodeURIComponent(booking.subject)}#booking`
```

This is a one-character-position fix: query param (`?subject=...`) must come BEFORE the hash (`#booking`) so that `useSearchParams().get('subject')` in BookingCalendar receives it. Browser URL parsing: fragment (`#`) terminates the search string — anything after `#` is the fragment, not a query param.
