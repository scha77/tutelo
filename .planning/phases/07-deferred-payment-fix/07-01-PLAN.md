<objective>
Fix the deferred booking flow so bookings accepted by the teacher before connecting Stripe are never permanently stuck at `pending` with no Checkout session.

Purpose: Two one-line query changes close a critical integration gap identified in the v1.0 audit. Without this fix, any teacher who accepts a booking request before completing Stripe Connect onboarding creates a booking that can never proceed to payment.

Output: `src/app/api/stripe/webhook/route.ts` with both `.eq('status', 'requested')` calls replaced by `.in('status', ['requested', 'pending'])`, plus two implemented Vitest tests covering the pending booking scenario.
</objective>
